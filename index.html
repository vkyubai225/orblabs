Our Stack

Orby,The Intent Engine
Orby, the Intent Engine
A Stripe-like integration that can be embedded into any app frontend to enable users from any chain to access the app. It can also be embedded in a wallet to enable users to interact with any app on any chain without fragmenting user accounts. Orby works by taking in a multi-chain interaction and converting it into an intent that can be executed on any chain via the Orb Stack.
Enables
Users to interact with any app they like on any chain.
App developers to build on the chain that is best for their apps instead of those with the most users.
Wallets to be universal from day one and automatically expand to new chains with every new deployment of the Orb Stack.
Chains to focus on attracting developers instead of users. With a single deployment of the Orb Stack, those developers all have access to all users in web3.

Orb Stack
05Universal Account System
Universal Account System
The topmost layer of the Orb stack. It enables apps and users to securely hold assets and perform transactions across chains without natively being present on those chains. It occasionally serves as the gateway through which users submit intents to the cross-domain intents protocol.
Enables
Cross-domain intent protocol
Wallets with web2 authentication
Multichain-wallets
Read developer docs
04Cross-Domain Intent Protocol
Cross-Domain Intent Protocol
An on-chain protocol that serves as the core of the Orb Stack. It can be called by anyone, including EOAs, to submit intents across chains. The protocol is responsible for verifying whether an intent has been satisfied and compensating solvers accordingly.
Enables
Chain-agnostic wallets
Chain-agnostic frontends
Frontends that enable every blockchain user to use your product regardless of chain

03Token Wrapping Protocol
Token Wrapping Protocol
A protocol that leverages the messaging framework to enable fluid movement of assets between chains primarily for computation. It is used by the cross-domain intent protocol for proper compensation of solvers.
Enables
Cross-domain intent protocols
Cross-chain meta transactions
Low cost transaction computations
E.g. have a rollup that is used just for offloading computations from more expensive chains
Cross-chain native applications
Multi-chain naming system
Cross-chain native tokens
Omnichain NFTs Omnichain FTs
Read developer docs
02Messaging Framework
Messaging Framework
An arbitrary message-passing layer that builds on our light clients to enable gas-efficient, secure, and fast message passing between different apps on different chains. It is used by the token wrapping protocol to send token information across chains.
Enables
Token Wrapping protocols
Cross-chain messaging platforms
E.g. create a messaging protocol that offers different trust guarantees, speed, etc for a particular use case
A wide range of cross-chain applications
Read developer docs
01Extensible Light Clients
Extensible Light Clients
The security foundation of the Orb Stack that enables apps to read data from any chain securely. Leveraged by our messaging framework to enable secure message passing across the Orb Stack. Can be embedded in other messaging protocols and frameworks to enable the same.
Enables
Secure messaging protocols
E.g. L2-L2 bridging in a trust-minimized manner
Secure cross-chain state accessors, i.e. services for accessing state of any blockchain
E.g. extend access to Ethereum services such as ChainLink oracles
Generalized co-processors
